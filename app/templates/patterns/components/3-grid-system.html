Our responsive grid system is comprised of twelve columns fluid with fixed 20px gutters, constrained within a 1200px container.

Columns

{% for j in [1, 2, 3, 4, 6] %} {{grid(j)}} {% endfor %}

Markup

The grid is constrained within a .container which gives the layout a maximum width. The grid itself represented by a .grid, with columns as a .grid__col. The column widths are expressed in terms of x of 12, prefixed by a breakpoint to allow different layouts at varying screen sizes. The breakpoints are "mobile first" meaning that they will be full-width by default. The following example demonstrates a layout which has two columns on the 'm' breakpoint and above.

{% filter markdown %} ```markup

This will be 6 columns wide.

This will be 6 columns wide.

``` {% endfilter %}

Reversible

A reversed grid allows you change the source order of the markup and still keep a left-to-right layout. As you can see in the example below, the columns appear in the reverse order that they occur in the markup:

{% filter markdown %} ```markup

1st in the DOM

2nd in the DOM

``` {% endfilter %}

Push and Pull

Columns can be pushed or pulled in a particular direction at a specified breakpoint. To reset a push at a subsequent breakpoint, use the nopush feature. The following example shows three columns all pushed to the right by one column at the m breakpoint. At the l breakpoint, the first column has its push removed.
